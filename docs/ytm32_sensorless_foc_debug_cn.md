## 1. 本次参数更新

本次已经根据你提供的数据更新了 [app/motor_user_config.h](/Users/major/work/1_Yuntu/M_Motor/5_YuntuMotorDemo/app/motor_user_config.h) 中的关键参数：

- 极对数：`4`

- 相电感 `Ls`：`0.00045158 H`
- 初始假设：`Ld = Lq = Ls`
- ADC参考电平：`5V`
- 母线分压：上拉 `4k`，下拉 `1k`
- 电流采样放大倍数：`10`

- 反电势测量：`AB线峰峰值 28.6V`，`电频 385Hz`

由反电势估算磁链的计算过程如下：

- 线电压峰值：`Vab_peak = 28.6 / 2 = 14.3V`
- 相电压峰值：`Vphase_peak = 14.3 / sqrt(3) = 8.256V`
- 电角速度：`omega_e = 2 * pi * 385 = 2419.03 rad/s`
- 磁链：`lambda = 8.256 / 2419.03 = 0.00341 Vs/rad`

当前配置里 `MOTOR_CFG_FLUX_LINKAGE_VS` 已按这个公式直接宏推导，不再写死常数。

## 2. 特别注意

- 现在 `ADC参考 = 5V`，`VBus分压比 = (4k + 1k) / 1k = 5`，所以母线电压最大可测值只有 `25V`。
- 这意味着实际调试时母线电压不要超过 `24V`，否则采样会饱和，电压保护和观测器都会失真。
- 如果你的系统准备跑 `24V` 以上母线，必须先改硬件分压，再继续调试。
- 我已经把自动起转关掉了：`MOTOR_APP_AUTO_START = 0`。第一次上电时这样更安全。
- 当前校准阶段也走 `eTMR -> TMU -> ADC0` 的硬件触发链，不再单独切软件触发。
- 校准时 `eTMR` 保持运行，PWM 影子寄存器预装为 `50%` 零矢量，占空更新点仍由 `CH1 VAL0 = 0` 触发；默认输出保持 mask，所以不会真正驱动功率管。
- 如果你发现 `ADC0_IRQHandler` 一直不进，第一优先检查三件事：
- `CIM->CTRL` 里的 `ADC0_TRIG_SEL` 是否已经切到 `TMU` 路径。
- `TMU` 是否已经把 `eTMR0_EXT_TRIG` 路由到 `ADC0_EXT_TRIG`。
- `eTMR0` 计数器是否真的启动，以及 `CH1 VAL0` 匹配事件是否在持续发生。

## 3. 建议调试顺序

### 第一步：只检查采样，不让电机转

- 上电前先把电源限流打开，建议从较低限流开始。
- 保持 `MOTOR_APP_AUTO_START = 0`，先下载程序但不要使能电机。
- 先确认 `MOTOR_STATE_OFFSET_CAL` 阶段能够自动累计采样并走到 `ALIGN`，这一步本身就能验证硬件触发链是通的。
- 用调试器查看 `MotorControl_GetStatus()` 返回值，确认：
- `bus_voltage_v` 与万用表实测接近。
- `phase_current_a/b/c` 在空载静止时接近 `0A`。
- 三相静止时电流偏置校准完成后，三相零漂不要明显偏大。

### 第二步：只验证 PWM 与对齐

- 先不要闭环跑速度。
- 可以临时把 `MOTOR_CFG_ALIGN_CURRENT_A` 保持当前较小值 `1A`。
- 手动调用 `MotorControl_Enable(true)`，先观察电机是否只出现轻微锁定动作，而不是剧烈抖动。
- 用示波器看三相 PWM，确认互补输出、死区、占空更新都正常。

### 第三步：验证开环起转

- 先保持较小的 `MOTOR_CFG_OPEN_LOOP_IQ_A = 1.5A`。
- 如果电机不能可靠起转，优先逐步增加：
- `MOTOR_CFG_ALIGN_CURRENT_A`
- `MOTOR_CFG_OPEN_LOOP_IQ_A`
- 每次建议只加 `0.2A ~ 0.5A`，不要一次跨太大。
- 如果起转时声音重、抖动大、相电流尖峰明显，先不要继续加电流，优先检查相序、采样极性和电角度方向。

### 第四步：验证观测器接管

- 开环阶段如果能稳定拉升，再看是否能从 `OPEN_LOOP_RAMP` 切到 `CLOSED_LOOP`。
- 重点观察：
- `electrical_speed_rad_s` 是否连续上升。
- `observer_locked` 是否拉高。
- `phase_error_rad` 是否逐渐收敛，而不是持续发散。
- 如果总是不能闭环，多半先看磁链、采样比例和电流方向。

### 第五步：再调速度环

- 电流环和观测器稳定后，再调 `MOTOR_CFG_SPEED_KP`、`MOTOR_CFG_SPEED_KI`。
- 速度环一定放到最后调，不要在电流环或观测器还不稳定时先调速度环。

## 4. 参数更新原则

### 4.1 极对数

- `极对数` 只影响机械量和电气量换算。
- 如果极对数错了，电角速度、RPM显示、速度环都会错，但电流环本体的数学结构没变。

### 4.2 Rs

- `Rs` 影响电流环和观测器。
- 冷机测得的 `Rs` 可以先用来起调，后续如果热机误差明显，可以再用热态值微调。
- `Rs` 偏小通常会让观测器和电流模型偏“冲”，偏大则响应会偏“钝”。

### 4.3 Ld / Lq

- 你当前只给了 `Ls`，所以首版按 `Ld = Lq = Ls` 处理。
- 这对表贴式永磁电机通常是合理的起点。
- 如果后续确认电机有明显凸极特性，再拆开单独调 `Ld` 和 `Lq`。

### 4.4 磁链

- 无传感器 FOC 里磁链是最关键参数之一。
- 你这次给的反电势数据很有价值，已经可以直接推导出第一版可用的 `flux linkage`。
- 如果后续表现为：
- 高速容易丢锁：优先检查磁链是不是偏小。
- 低速角度乱跳：优先检查磁链范围和电流采样噪声。
- 一般建议 `LAMBDA_MIN/MAX` 先围绕估算值放在大约 `70% ~ 140%` 的范围内，再实机细调。

### 4.5 电流采样链路

- 电流换算公式是：
- `I = ADC_voltage / (Rshunt * Gain)`
- 当前条件下：
- `Rshunt = 0.005`
- `Gain = 10`
- 所以 `0.05V/A`
- 这组参数只要错一个，实际电流显示和保护阈值都会一起错。

### 4.6 VBus 分压

- 母线换算公式是：
- `Vbus = Vadc * (Rtop + Rbottom) / Rbottom`
- 当前分压比就是 `5`
- 由于 ADC 满量程 5V，所以 VBus 满量程只有 `25V`
- 这个参数必须和硬件实际完全一致，否则欠压/过压保护一定不准。

## 5. 实机调试注意事项

- 第一次带电调试一定用限流电源。
- 先空载，再半载，最后再带真实机械负载。
- 第一次闭环前，建议把负载尽量减小，避免因为启动电流不够误判成算法问题。
- 如果电机一使能就强烈抖动，优先检查三件事：
- 电流采样极性是否反了。
- 相序和PWM相位映射是否一致。
- 电角度正方向是否和实际旋转方向一致。
- 如果电机能转但速度估算明显漂，优先检查：
- `flux linkage`
- `Rs`
- VBus采样是否准确
- 如果闭环切入瞬间扭矩台阶明显，可以适当拉长 `MOTOR_CFG_CLOSED_LOOP_BLEND_MS`。

## 6. 建议的首轮实机策略

- 先保持当前安全参数上电：
- `ALIGN_CURRENT_A = 1.0A`
- `OPEN_LOOP_IQ_A = 1.5A`
- `MAX_IQ_A = 6.0A`
- `PHASE_OVERCURRENT_A = 10.0A`
- 如果电机完全拉不动，再逐步增加对齐电流和开环电流。
- 如果能起转但闭环失败，先不要继续加电流，先看观测器相关参数。
